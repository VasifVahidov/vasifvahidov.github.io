So I was reading through various German news sources last week, and I had this moment where I realized I was consuming article after article about Bundestag decisions, but I had absolutely no clue how any of it actually affected me or anyone I know. Like, great, the coalition passed some new law—but does that mean my rent goes up? Do I need to change how I file taxes? And where exactly does this even fit on the political spectrum?

That's when I decided to build something about it instead of just wondering. But not just any news analyzer—I wanted something that could automatically pull articles from publicly available RSS feeds and tag them with WHO gets affected, HOW they're affected, WHERE it happens, and even what political lean the content might have. Most importantly, I wanted to explore semantic similarity search using embedding vectors—a fascinating ML technique that can find meaningful connections between different pieces of text.

The Problem That Actually Motivated Me

Most news platforms are terrible at explaining impact. They'll tell you what happened, maybe even why, but rarely who gets affected and how badly. I wanted something that could automatically process articles from publicly available news feeds, then output: "This affects students in Bavaria by potentially increasing their tax refunds by €200" along with contextual information about the political perspective.

Key Insight

Specific. Actionable. Automatically updated. Not just "economists say this could have implications." Plus, this seemed like a perfect educational project to learn about semantic similarity search, embedding vectors, and automated content analysis—skills I could apply to many other projects later.

Why I Went Full Open Source

I could have just used GPT-4 and called it a day, but where's the fun in that? Plus:

  • I'm a broke grad student
  • I wanted to actually learn something
  • Vendor lock-in gives me anxiety
  • Free stuff is better stuff

So I decided to build the whole thing using only free, open-source tools. Turns out, it's totally doable and incredibly educational.

How Semantic Similarity Search Actually Works

The really exciting part of this project was implementing semantic similarity search using embedding vectors. This is a powerful technique that can find meaningful connections between different pieces of text, even when they use completely different words.

Core Concept

Instead of matching exact keywords, we convert text into high-dimensional vectors that capture semantic meaning. Similar concepts cluster together in this vector space, allowing us to find articles about "student financial aid" even when searching for "university cost support."

const PUBLIC_RSS_FEEDS = [
  'https://example-news-site.com/rss',
  'https://another-public-feed.com/xml/rss',
  // Add any publicly available RSS feeds
];

async function fetchPublicNewsFeeds() {
  const articles = [];
  
  for (const feedUrl of PUBLIC_RSS_FEEDS) {
    try {
      // Use a CORS proxy for educational purposes
      const response = await fetch(`/api/rss-proxy?url=${encodeURIComponent(feedUrl)}`);
      const rssData = await response.text();
      const parser = new DOMParser();
      const xmlDoc = parser.parseFromString(rssData, 'text/xml');
      
      const items = xmlDoc.getElementsByTagName('item');
      for (let item of items) {
        const article = {
          title: item.querySelector('title')?.textContent,
          content: item.querySelector('description')?.textContent,
          url: item.querySelector('link')?.textContent,
          pubDate: item.querySelector('pubDate')?.textContent
        };
        
        // Clean HTML from description for analysis
        article.content = cleanHtmlContent(article.content);
        articles.push(article);
      }
    } catch (error) {
      console.log(`RSS feed temporarily unavailable:`, error);
    }
  }
  
  return articles;
}

Python: The Semantic Search Engine

Python 
from sentence_transformers import SentenceTransformer, util
import json
import torch
import numpy as np

# Using multilingual model for semantic similarity across languages
model = SentenceTransformer("paraphrase-multilingual-MiniLM-L12-v2")

def perform_semantic_similarity_search(article_text):
    """
    This is the core of semantic similarity search using embedding vectors.
    Instead of keyword matching, we find conceptually similar content.
    """
    
    # Load our curated impact patterns (this is the knowledge base)
    with open("data/impact_patterns.json", "r", encoding='utf-8') as f:
        impact_patterns = json.load(f)
    
    # Convert all patterns to embedding vectors (cache this in production)
    pattern_texts = [pattern['text'] for pattern in impact_patterns]
    pattern_embeddings = model.encode(pattern_texts, convert_to_tensor=True)
    
    # Convert the input article to an embedding vector
    article_embedding = model.encode(article_text, convert_to_tensor=True)
    
    # Compute cosine similarity between article and all patterns
    # This is where the magic happens - finding semantic similarity!
    similarities = util.pytorch_cos_sim(article_embedding, pattern_embeddings)[0]
    
    # Get the top 3 most similar patterns
    top_matches = torch.topk(similarities, k=3)
    
    results = []
    for score, idx in zip(top_matches.values, top_matches.indices):
        pattern = impact_patterns[idx.item()]
        results.append({
            'who': pattern['affected_group'],
            'how': pattern['impact_type'], 
            'where': pattern['location'],
            'confidence': score.item(),
            'pattern_text': pattern['text']
        })
    
    return results

This semantic similarity approach is incredibly powerful because it can match concepts like:

  • "Students face financial pressure" ↔ "University attendees struggle with costs"
  • "Small businesses see tax relief" ↔ "Entrepreneurs benefit from reduced fiscal burden"
  • "Munich residents experience housing issues" ↔ "Local citizens face accommodation challenges"

What I Learned About Semantic Search

  • Embedding vectors are genuinely magical: The way sentence-transformers can find conceptual connections between completely different phrasings is mind-blowing.
  • JSON-based pattern matching scales surprisingly well: Instead of training a custom model, I curated a JSON database of impact patterns.
  • Multilingual embeddings open up global possibilities: The paraphrase-multilingual model handles German-English concept matching beautifully.
  • RSS parsing is still a valuable skill: Despite being "old tech," RSS feeds remain reliable for accessing structured content.
  • Cosine similarity is the unsung hero: The math behind finding similar vectors is surprisingly elegant.

Try It Yourself

I've made the project available for educational purposes. The setup is designed to be learning-friendly:

Installation 
git clone [repo-url]
cd capycapynews-educational

# Install Python dependencies for semantic analysis
pip install -r requirements.txt

# Download the multilingual sentence transformer model
python utils/download_models.py

# Start the educational RSS proxy server
node api/rss-proxy.js &

# Serve the frontend (any simple HTTP server works)
python -m http.server 8000
# OR
npx serve .
